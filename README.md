C Shell :
I implements a custom C Shell in C, designed a wide range of shell functionalities. The shell displays a prompt in the format <Username@SystemName:current_path> where the home directory is abbreviated as ~, accepts user input, and parses commands according to a specified context-free grammar, printing “Invalid Syntax!” for invalid commands. It supports built-in intrinsics such as hop for changing directories, reveal for listing files with optional flags -a and -l, and log for maintaining a persistent history of the last 15 commands with options to display, purge, or execute a previous command. The shell also allows execution of arbitrary commands, input and output redirection using <, >, and >>, and piping using |. Sequential execution of commands using ; and background execution with & are fully supported. Advanced job control features include activities to list running or stopped processes, ping to send signals to processes, and handling of keyboard shortcuts such as Ctrl-C to send SIGINT to foreground processes, Ctrl-D to terminate the shell and kill all child processes, and Ctrl-Z to stop foreground processes and move them to the background. The shell also supports fg and bg commands to bring background or stopped jobs to the foreground or resume them in the background. The project is implemented in C using only POSIX-compliant functions and libraries, compiled with GCC using strict POSIX flags to ensure standard compliance. The shell can be compiled with make all and executed with ./shell.out, providing a fully functional and interactive environment for command execution, job control, and process management
